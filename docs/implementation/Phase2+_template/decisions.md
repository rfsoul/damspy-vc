## Editing Guidelines (Do Not Modify Below This Line)

Record important technical or architectural decisions.

Each entry should briefly describe:

• the decision that was made
• the reasoning behind it
• any alternatives that were considered

A typical entry might include:

### Decision: <short title>

**Decision**

Describe the choice that was made.

**Reasoning**

Explain why this option was chosen.

**Alternatives Considered**

List other options that were evaluated.

**Consequences**

Describe any important effects or trade-offs resulting from the decision.


This document records **important technical and architectural decisions** made during the life of the project.

Its purpose is to preserve the **reasoning behind decisions**, so future contributors understand why the system was designed a certain way.

Use this document for:

• architecture choices
• technology selections
• major design trade-offs

Avoid using this document for:

• implementation details
• step-by-step engineering work
• future ideas or feature requests

Engineering tasks belong in `liveplan.md`.
Ideas for possible future work belong in `futuredirections.md`.

Only edit the **Content section above** unless the documentation system itself is being changed.
