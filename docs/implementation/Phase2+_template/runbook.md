## Editing Guidelines (Do Not Modify Below This Line)

This document describes **how to operate and troubleshoot the system**.

It is intended for maintainers who need to run, monitor, or recover the system during development or production operation.


Typical sections may include:

### System Startup

Describe how the system is started in a development or production environment.

### Normal Operation

Describe what normal system behavior looks like.

Include important background tasks such as:

• scheduled jobs
• message processing
• event triggers

### Monitoring

Describe how to confirm the system is operating correctly.

Include indicators such as:

• logs
• health checks
• queue activity
• expected outputs

### Troubleshooting

Document common operational problems and how to diagnose them.

Include:

• symptoms
• likely causes
• steps to investigate
• recovery actions

### Recovery Procedures

Describe how to recover the system from failures such as:

• service crashes
• lost connections
• failed integrations


Use this document to record:

• operational procedures
• system monitoring practices
• troubleshooting steps
• recovery actions

Avoid including:

• architecture explanations
• engineering design decisions
• implementation details

Architecture belongs in `architecture.md`.
Design decisions belong in `decisions.md`.

Only edit the **Content section above** unless the documentation system itself is being changed.
